ATIVIDADE 1

A realização de boas práticas para a criação de um código limpo é fundamental para o sucesso. Alguns exemplos de boas práticas seriam:

- Manter-se sempre atualizado com o surgimento de novas tecnologias porque o frontend sempre está em evolução;

- Garantir a responsividade do site para diversos dispositivos com diferentes tamanhos de telas;

- Utilizar sistemas/aplicações para o versionamento do código e auxiliar na colaboração em equipe. Os principais exemplos são: Git e GitHub.

Existem também alguns exemplos para se ter um código limpo:

Exemplo 1:

Criar um arquivo CSS separado do HTML para a estilização do site é um exemplo de código limpo, porque assim você mantém o código mais organizado (sendo mais fácil para corrigir alguns erros ou até mesmo ajudar os seus colaboradores a compreenderem o seu código).

Exemplo 2:

A utilização do React é um outro exemplo para a criação de um código limpo, porque ao invés de codificar todos os componentes em uma mesma página, se cria um arquivo.jsx para cada um e depois eles são exportados para um arquivo principal (no meu caso: main.jsx), dessa forma o código torna-se mais organizado, permitindo que caso ocorra algum erro em específico de um determinado componente, basta analisar o seu respectivo arquivo.

O que geralmente não torna um código limpo são práticas ruins como falta de modularidade, ou seja, não é criado arquivos específicos separados (Componentes ou CSS), criação de funções extremamente longas, confusas e desnecessárias, criar nomes de variáveis incoerentes ou que não tenham sentido e entre outras práticas.
